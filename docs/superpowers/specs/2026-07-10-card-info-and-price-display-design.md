# 카드 정보·가격 표시 개편 설계

> 2026-07-10 · 브레인스토밍 산출물
> 관련: `DESIGN.md`(팔레트·타이포), `docs/superpowers/specs/2026-07-09-seasonal-picks-design.md`(원설계)

## 배경

현재 카드는 한 줄 텍스트로 가격을 보여준다(`18,200원/10개 · 한 달 전보다 26% ↓`).
"이번 달 제철 중 값이 내려온 것"을 알려주는 앱의 정체성에 비해 **가격의 선명함**과
**계절 정보의 깊이**가 약하다. 톤(담백·손그림·쪽빛 규율)은 유지하되, 가격을 전단지처럼
또렷하게 하고 이미 가진 데이터(`priceMonthAgo`·`priceYearAgo`·`peakMonths`·`whyNow`·
`seasonMonths`·`category`)를 카드에 살려낸다.

이 설계는 **API 키 없이 구현 가능**하다 — 필요한 값이 모두 현행 데이터 모델에 있다.

## 원칙 (유지)

- 런타임 의존성 0, 런타임 외부요청 0. 인터랙션은 CSS 우선(JS 최소).
- 사용자 문구는 담백한 한국어. 이커머스 화법 금지.
- 웜 컬러는 배경/칩으로만, 글자·강조는 쪽빛(`--ink`). (상승색은 예외 — 아래 개정)
- 카드 안에 일러스트는 여전히 없음. 새로 들어오는 건 **정보 그래픽**(볼드 가격·칩·
  스파크라인·필터)이지 장식이 아니다.

## 범위

원안 4 + 아이데이션 6, 총 10개. 밀도를 위해 카드는 `<details>` 접힘/펼침으로 나눈다.

| # | 항목 | 위치 |
|---|---|---|
| 원안-1 | 과일/채소 분류 | 상단 필터 토글(CSS) |
| 원안-2 | 이모지 유지 | 접힘 |
| 원안-3 | 전단지처럼 볼드한 가격 | 접힘(가격 블록) |
| 원안-4 | 지난값 실제 표기 + 화살표 | 접힘(가격 블록) |
| 제안-1 | 작년 대비 신호 | 펼침(스파크라인) |
| 제안-2 | "지금 절정" 표시 | 접힘(이름 옆 dot + 툴팁) |
| 제안-3 | whyNow 노출 | 펼침 첫 줄 |
| 제안-4 | 개당값 | 접힘(가격 블록 하단) |
| 제안-5 | 정직한 빈 상태 | 리스트 대체 문구 |
| 제안-6 | "곧 제철"(다음 달) | 하단 제철 섹션 |

## 카드 구조

`<details class="card" data-cat="fruit|vegetable">` 유지. summary=접힘, 나머지=펼침.

### 접힘 (summary) — 좌/우 2단

```
[이모지]  이름  ·(절정 dot)              지난값(취소선)
          품종 · 단위                [화살표 칩 %]  큰 볼드 가격
                                              개당값
```

- **왼쪽**: 이모지 + 이름(+ 절정 dot) + 품종·단위(`kindName` · `unit`)
- **오른쪽**(우측 정렬, 세로):
  1. 취소선 지난값 (`priceMonthAgo`, "지난 달 대비" 등 라벨 없음 — 취소선으로 충분)
  2. 화살표 칩(등락 %) + 큰 볼드 현재가 (`price`)
  3. 개당값 보조줄

기존 텍스트 배지("제철 한창"/"제철")와 한 줄 `formatPrice`는 제거하고 위 블록으로 대체.
whyNow는 접힘에서 빼고 펼침으로 옮긴다.

### 펼침

1. **whyNow 한 줄** — `whyNowLine(profile, month)`, 쪽빛, 편집자 목소리
2. **가격 스파크라인** — 작년 이맘때 / 한 달 전 / 지금 (아래 명세)
3. **장보기 노트** — 고르는 법 / 보관 / 쓰임 (아래 명세)

## 가격 블록 명세

색은 등락 방향으로 가른다. **이 앱에서 하락은 좋은 소식이므로 쪽빛(차분),
상승은 러스트(주의)** 로 둔다. 칩은 "배경 저채도 + 글자 고채도"로 내부 채도 대비를 준다.

| 방향 | 큰 가격 | 칩 배경 | 칩 글자·화살표 |
|---|---|---|---|
| 하락 (`pct < 0`) | `--ink` | `--tint` (저채도 노랑) | `--ink` (고채도 쪽빛) |
| 상승 (`pct > 0`) | `--rise` | `--rise-lo` (저채도 크림) | `--rise` (고채도 러스트) |
| 변동 미미(<1%) | `--ink` | 칩 생략, "한 달 전과 비슷" 보조 문구 | — |

- **화살표**: 채우기 없이 획만 있는 SVG(`stroke:currentColor`), 손그림 톤. 하락=아래,
  상승=위 화살표.
- **취소선 지난값**: `priceMonthAgo`가 `null`이면 생략.
- **개당값**: `unit`이 셀 수 있는 복수(예: "10개")면 `price / 개수`를 "개당 N원"으로.
  단수("1개")·무게("1kg") 단위면 생략. (파싱 규칙은 구현 세부 — 선행 숫자+단위 추출)

## 스파크라인 명세 (펼침)

`priceYearAgo` · `priceMonthAgo` · `price` 3점. dataviz 가이드 적용:
자리(magnitude/시간축), **단일 계열 → 범례 없음**, "지금"만 강조, 나머지는 물러남.

- 인라인 SVG, **균일 스케일**(`preserveAspectRatio` 기본값 — `none` 금지: 원 찌그러짐).
  `viewBox="0 0 300 72"`, `width:100%`.
- x = 시간순(작년 이맘때 → 한 달 전 → 지금), 3점 균등 배치(x=45,150,255).
- y = 값(높을수록 위). 3점 min/max로 정규화.
- 요소: 얇은 추세선(`--muted`, 1.1, opacity .65) · 점(과거 r 1.9/테두리 0.8,
  지금 r 2.3 `--ink` 채움) · **값은 점 위**, **라벨은 점 아래** · 그 사이 아주 가는
  x축(`--axis` #CBC2B0, 0.3px). 축 위 8px / 아래 8px 여백.
- 접근성: `<svg aria-label="가격 추이: 작년 이맘때 X · 한 달 전 Y · 지금 Z">`.
- 구현 시 dataviz `validate_palette.js`로 팔레트 확인(라이트/다크).
- 확정 목업: `.superpowers/brainstorm/231-1783688577/content/loaded-card-v8.html`

## 장보기 노트 명세 (펼침)

기존 `<dl>` 상세를 "공책" 스타일로. **배경 채우기 없음**(먹지 느낌):

- 왼쪽 세로 여백선(`::before`, `--margin` #E3C4B8, 1px)
- 각 줄 밑 점선 괘선(`1px dotted --line`), 마지막 줄은 없음
- **키 고정 너비 정렬**: `grid-template-columns: 4rem 1fr`. 키(`고르는 법`/`보관`/
  `쓰임`)는 `--ink` 볼드, 설명은 둘째 열에서 줄바꿈. 라벨 문구는 기존
  "고르는 법 / 보관법 / 이렇게 먹어요"에서 "고르는 법 / 보관 / 쓰임"으로 축약(정렬 안정).

## 절정 dot + 툴팁 (접힘)

- `inPeak`(=`peakMonths.includes(month)`)면 이름 옆에 `--accent` 점 하나
  (`box-shadow`로 `--tint` 링). 텍스트 없음.
- `<button>`로 감싸 hover/focus 시 툴팁("지금이 맛의 절정이에요"). 모바일은 탭=focus로
  표시 → **JS 불필요**(`:hover, :focus`). 툴팁은 `--ink` 배경·흰 글자.

## 과일/채소 필터 (상단, CSS-only)

- 라디오 3개(`전체`/`과일`/`채소`) + `label` 칩. 단일 순위(정렬)는 안 건드리고 **표시만**
  토글: `#f-fruit:checked ~ .list .card[data-cat=vegetable]{display:none}` 식.
- 칩 스타일: 선택 = `--tint` 배경 + `--ink` 글자, 비선택 = `--line` 테두리 + `--muted`.
- JS 0. `<details>`·툴팁과 같은 "JS 없는 인터랙션" 철학.
- 카드에 `data-cat` 속성 추가(값은 `profile.category`).

## 빈 상태 / 곧 제철

- **빈 상태(제안-5)**: 현행은 "제철 0개"만 처리. 여기에 **"제철은 있으나 유의미하게
  내려온 게 없음"** 케이스를 더해 "이번 주는 크게 내려온 게 없어요. 제철은 그대로
  곁에 있어요." 같은 담백한 문구. (판정 기준: 하락 픽이 0개일 때)
- **곧 제철(제안-6)**: 하단 "N월의 제철" 섹션 옆/아래에 다음 달 새로 철 드는 품목
  (`seasonMonths.includes(month+1) && !includes(month)`) 한 줄 예고. 순수 파생, 새 데이터 없음.

## 데이터·타입 변경

- `PriceView` 확장: `priceMonthAgo: number | null`, `priceYearAgo: number | null` 추가.
  `priceView()`에서 `entry`의 값을 통과. (`price`·`unit`·`changeVsMonthAgoPct`는 유지)
- `render.ts`: `formatPrice`(한 줄)·`renderCard`(배지/summary/dl) 전면 개편.
  새 함수 후보: `renderPriceBlock`, `renderSparkline`, `renderNote`, `renderPeakDot`.
- `picks.ts` `selectPicks` 정렬 로직은 유지(절정 먼저 → 하락 큰 순). 필터는 표시 계층.
- `data.ts` 접근 경계 유지 — UI가 JSON 경로를 직접 알지 않는다.

## DESIGN.md 개정 사항 (구현 시 반영)

| 토큰 | 변경 | 이유 |
|---|---|---|
| `--rise` | `#8C6A5D`(밤색) → `#C0392B`(러스트) | 밤색은 저채도라 칩 내부 채도 대비가 안 남 |
| `--rise-lo` | 신규 `#EFEAE3` | 상승 칩 저채도 배경 |
| `--margin` | 신규 `#E3C4B8` | 장보기 노트 여백선 |
| `--axis` | 신규 `#CBC2B0` | 스파크라인 x축 |

- "규율" 문단: 상승색이 밤색→러스트로 바뀌었음을 반영. "먹는 웜 컬러가 글자를 싣지
  않는다"는 유지하되, `--rise`/`--ink`는 **의미 마커**로서 글자에 실릴 수 있음을 명시.
- "카드 안엔 일러스트 없음" 항목: 정보 그래픽(볼드 가격·칩·스파크라인·필터)은 장식이
  아니므로 허용됨을 한 줄로 명확화.

## 비범위 / 열린 질문

- **KAMIS OpenAPI 전환**: 별개 작업. 현재 `dailyPriceByCategoryList` 유지. 등락 판정
  단순화가 필요하면 `dailySalesList`(당일/1일·1개월·1년 전) 검토 — 이 설계와 독립.
- 다크 모드: 현재 라이트 전용. 스파크라인/칩 색은 다크 대응 시 별도 검증 필요.
- "지난 주" 비교: 현행 데이터엔 주간 값이 없어 "한 달 전"만. API 전환 시 확장 여지.
