# 구조적 디자인 토큰 — 설계

*색은 이미 토큰화돼 있으나(`:root`의 `--ink`·`--muted`… + 계절 시스템), 간격·폰트패밀리·
라운드는 리터럴이 흩어져 시스템이 없다. 이를 이름 있는 토큰으로 정리한다. 근거: impeccable
(layout) 감사 — `src/style.css` 425줄에 간격 ~31종·폰트 스택 7곳·라운드 4종.*

작성: 2026-07-17. 범위: **구조적**(폰트패밀리·라운드·간격). **타입 스케일(font-size)은 범위 밖 — 다음 작업.**

## 왜

DESIGN.md가 색·폰트 역할·팔레트를 정하지만, 실제 CSS는 간격을 매번 새 리터럴로 정한다
(감사: margin/padding 값 ~31종, 0.06rem부터 3rem까지 거의 연속). 조사일 헤더에서 본 "0.42 vs
0.45 vs 0.48을 매번 새로 판단"하는 그룹핑 사고가 파일 전반에서 반복된다. 이름 있는 스케일로
접으면 (1) 그 판단을 한 번만 하고 (2) 미래 변경이 스케일 위에서 일관되며 (3) DRY.

detect.mjs가 이 문제를 못 잡은 건 값이 `rem`이라 "임의 픽셀"로 안 걸려서지, 425줄에 31종은
시스템이 아니다.

## 결정 (확정 — 브레인스토밍에서 사용자 승인)

- **간격: 리듬 보존.** 앱의 사실상 0.4rem 계열 리듬을 살려 스냅. 정석 4pt로 재편하지 않는다
  (픽셀 이동 최소화 우선 — 이미 사인오프된 UI라 정체성 보존).
- **1.0rem 단계 포함(9단계).** 감사에서 스페이싱 `0.95·1.0·1.05·1.1` = **12곳**이 앱의 최대
  클러스터다. 1.0 단계가 없으면 이 12곳이 0.8/1.2로 3~4px씩 밀려 리듬 보존 취지에 반한다.
- **시맨틱 이름** (`--space-xs` … 계열, `--spacing-8` 같은 숫자명 아님).
- **폰트패밀리·라운드도 토큰화** (시각 변화 ≈0).
- **전면 적용.** 토큰만 정의하고 안 쓰면 무의미 — `style.css`의 간격·폰트·라운드 리터럴을
  토큰 참조로 교체한다.
- **색·계절·그림자(`--lift`)는 손대지 않는다.**

## 토큰

기존 색 토큰이 사는 `:root`에 추가한다.

### 간격 스케일 (9단계)

```css
--space-3xs: 0.2rem;
--space-2xs: 0.3rem;
--space-xs:  0.4rem;
--space-sm:  0.6rem;
--space-md:  0.8rem;
--space-lg:  1.0rem;
--space-xl:  1.5rem;
--space-2xl: 2.5rem;
--space-3xl: 3.0rem;
```

스냅 매핑 (현재 값 → 토큰; 이동은 root 16px 기준):

| 토큰 | 값 | 흡수 (현재 값) | 최대 이동 |
|---|---|---|---|
| `--space-3xs` | 0.2rem | 0.06·0.1·0.18·0.2 | +2.2px (0.06, 1곳) |
| `--space-2xs` | 0.3rem | 0.25·0.3·0.32·0.34·0.35 | ≤0.8px |
| `--space-xs` | 0.4rem | 0.4·0.42·0.45·0.48 | ≤1.3px |
| `--space-sm` | 0.6rem | 0.5·0.55·0.6 | +1.6px (0.5) |
| `--space-md` | 0.8rem | 0.75·0.8·0.85 | ≤0.8px |
| `--space-lg` | 1.0rem | 0.95·1·1.05·1.1·1.2 | 1.2→1.0 −3.2px |
| `--space-xl` | 1.5rem | 1.35·1.4·1.5·1.6 | ≤1.6px |
| `--space-2xl` | 2.5rem | 2.2·2.5 | +4.8px (2.2) |
| `--space-3xl` | 3.0rem | 3 | 0 |

- **경계값 반올림 규칙:** 가장 가까운 단계로. 정확히 중간이면(예: 0.5, 1.2) **분리를 지키는 쪽**
  (더 큰 간격을 만드는 쪽)으로 — 여백은 줄이는 것보다 유지가 안전. 단 위 표의 확정 매핑이 우선.
- `gap` 값(0.2·0.4·0.45·0.5·0.55·0.6·0.9·1.25·1.4rem)도 같은 스케일로 스냅.
  0.9→1.0(`lg`), 1.25→1.5(`xl`) — 1.25는 1.0과 1.5의 정확한 중간이라 경계 규칙(분리를 지키는 큰 쪽)에 따라 `xl`.
- 헤더 스페이싱(직전 작업, 0.45/1.4rem 등)도 토큰으로: `header h1` 하단 0.45→`--space-xs`(0.4),
  `header` 하단 1.4→`--space-xl`(1.5). **이동 확인:** 0.45→0.4는 −0.8px, 1.4→1.5는 +1.6px —
  방금 사인오프한 3px/7px/23px 리듬이 3px/6px/24px로 미세 변동. 브라우저에서 재확인한다.

### 폰트 패밀리 토큰 (시각 변화 0)

```css
--font-display: 'MaruBuri', serif;
--font-hand: '학교안심 받아쓰기 L', serif;
--font-body: 'Apple SD Gothic Neo', 'Malgun Gothic', 'Noto Sans KR', sans-serif;
```

- 7곳 인라인 `font-family` 선언을 위 토큰 참조로 교체. **값은 현재 스택과 바이트 동일**하게
  옮긴다(폴백 순서·따옴표 포함) — 순수 DRY, 렌더 무변화.
- `--font-hand`의 정확한 스택은 현재 `.why` 규칙에서 그대로 복사(DESIGN.md: 못 구하면 `serif` 폴백).

### 라운드 토큰

```css
--radius-crisp: 0.2rem;  /* 메모 낱장 카드 — DESIGN.md "crisp" */
--radius-soft: 0.4rem;   /* 칩·배지 */
--radius-pill: 1rem;     /* 알약형 */
```

- 현재 4값(0.15·0.2·0.4·1rem) → 3토큰. **0.15→0.2**(`crisp`)로 통합(−0.8px, 근접 중복 제거).
  0.4→`soft`, 1→`pill`. 실제 어느 요소가 어느 토큰인지는 구현 시 규칙별로 판정(카드=crisp,
  칩/배지=soft, 알약=pill) — 표기값이 아니라 **역할**로 고른다.

## 적용 원칙

- 리터럴 → `var(--token)` 교체는 **의미가 맞는 곳**에만. 예: 스파크라인 x축 좌표, 폰트 두께,
  선 두께(1px), opacity, `--lift` 그림자 오프셋 등 **간격이 아닌 수치는 건드리지 않는다.**
- 한 선언에 복합 margin(`0.2rem 0 1.3rem`)이면 각 값을 개별 토큰으로: `var(--space-3xs) 0 var(--space-lg)`.
- 토큰으로 못 접는 진짜 예외값이 있으면(디자인상 꼭 그 값이어야 하는 곳) 리터럴로 두고 **한 줄
  주석으로 왜 스케일 밖인지** 남긴다(침묵하지 않는다).

## 검증 (완료 게이트)

- **게이트 = `npm test` 와 `npx tsc --noEmit` 둘 다.** (CSS 변경이라 로직 영향은 없지만 규칙.)
- **브라우저 재실측 + 스크린샷 사인오프** — 간격 스냅이 픽셀을 미세 이동시키므로 주요 화면을
  실제로 연다: 메인(`/`) 헤더·필터·카드 리스트·카드 펼침(레시피 핀메모)·`/coming`·빈 상태.
  폰트·라운드는 시각 무변화라 스팟 확인만.
- **회귀 없음 확인:** 토큰화 전/후 스크린샷 비교로 "의도한 미세 이동" 외의 변화(레이아웃 깨짐·
  마진 붕괴 변화)가 없는지 본다. 특히 복합 margin 분해 시 `margin-collapse` 거동 유지.
- **감사 재실행:** `node .claude/skills/impeccable/scripts/detect.mjs --json --scope layout src/style.css`로
  스냅 후 임의값이 안 남았는지(스케일 밖 리터럴은 주석 처리됐는지) 확인.

## 범위 밖 (하지 않는다)

- **타입 스케일(font-size).** 18종을 6단계로 접는 별도 작업. 글자 크기 변동이 크고 시각 영향이
  간격보다 커 분리한다.
- **색·계절 팔레트·`--lift`.** 이미 토큰화됐거나 DESIGN.md 확정 — 재편 안 함.
- **컴포넌트 추출.** 이번은 값 토큰만. 반복 UI 패턴의 컴포넌트화는 별개.
- **간격이 아닌 수치**(선 두께·opacity·좌표·폰트 두께).
