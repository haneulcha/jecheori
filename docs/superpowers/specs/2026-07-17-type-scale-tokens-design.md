# 타입 스케일 토큰 — 설계

*구조적 토큰(간격·폰트패밀리·라운드, `2026-07-17-design-tokens-structural`)에서 의식적으로
미룬 축. `font-size`가 ~18종(0.68–1.7rem) 거의 연속 램프라 시스템이 없다. 이를 이름 있는
6단계 스케일로 접는다. 근거: impeccable 감사 — `src/style.css`.*

작성: 2026-07-17. 범위: **`font-size`만**. `line-height`·`letter-spacing`·`font-weight`는 범위 밖.

## 왜

DESIGN.md가 타입 역할(디스플레이=마루부리 h1·절기 / 손글씨=받아쓰기 `.why` / 본문=시스템 /
라벨 / 숫자)을 정하지만, 실제 크기는 매번 새 리터럴이다(0.68·0.72·0.74·0.75·0.78·0.79·0.8·
0.82·0.85·0.88·0.9·0.95·0.98·1·1.05·1.3·1.5·1.7). 간격을 스케일로 접은 것과 같은 정리다 —
크기 판단을 한 번만 하고, 위계가 일관되며, DRY.

간격보다 **글자 크기 이동이 눈에 띄므로** 리듬 보존(시그니처 무이동·본문 클러스터만 ≤0.8px)을
택했고, 브라우저 사인오프 단계가 더 중요하다.

## 결정 (확정 — 브레인스토밍에서 사용자 승인)

- **리듬 보존, 6단계.** 정석 모듈러(×1.2)로 재편하지 않는다 — 시그니처 크기까지 크게 움직이는 걸 피한다.
- **시그니처 무이동:** 아이브로 `.week` 0.9 · h1 1.5 · 디스플레이(이모지·큰가격) 1.7 은 자기 단계를
  정확히 갖는다(DESIGN.md "시그니처는 머리말, 나머지는 조용히").
- **각 단계는 역할을 갖는다.** t-shirt 이름 옆에 그 단계가 담는 역할을 주석으로 못박아, 미래에
  "새 라벨은 몇 rem?"을 값이 아니라 역할로 판단하게 한다(아래 토큰 블록).
- **단일 사용 단계(1.3rem) 제거 → ‹ › 넘김 셰브론(`.nav`)을 `--text-lg`(1.5)로 흡수한다.** 원래
  1.3은 레시피 핀메모의 ‹ › 넘김 버튼 한 곳뿐이라 전용 단계가 냄새였다. h1과 같은 디스플레이 티어로
  올려 6단계로 단정하게 한다. **이 한 곳(‹ › 셰브론)만 의도적으로 +3.2px 커진다**(리듬 보존의 유일한
  예외 — 사용자 결정). (`.empty/.loading` 빈 화면은 0.95→`md`로 평범한 +0.8px 스냅이다, 별개.)
- **`font-size`만** 토큰화. `line-height`·`letter-spacing`·`font-weight`는 별개 축이라 안 건드린다.
  (leading을 미뤄도 안전한 근거는 아래 "적용 원칙" 참고 — 모든 line-height가 단위 없는 값이다.)
- **`.spark`의 `9px`(SVG 데이터 라벨) 제외** — 차트 내부 마이크로 라벨, 텍스트 토큰이 아니다.
- 색·간격·폰트패밀리·라운드 토큰은 그대로.

## 토큰

기존 토큰이 사는 `:root`에 추가한다.

```css
--text-2xs: 0.7rem;  /* 마이크로 라벨·캡션 (레시피 라벨·영양 serving·메모 카운트·절정 툴팁) */
--text-xs:  0.8rem;  /* 소형 라벨 (품종·근접 문구·칩·영양표 행·푸터·목차 제목) */
--text-sm:  0.9rem;  /* 보조 텍스트·절기 아이브로 (아이브로·필터 라벨·취소선 가격·nodrop) */
--text-md:  1rem;    /* 본문·카드/섹션 제목 (본문·카드 타이틀·원 단위·메모 h3·다가오는 달 h2) */
--text-lg:  1.5rem;  /* 디스플레이 헤딩 — h1 (+ ‹ › 넘김 셰브론 1.3→1.5 흡수) */
--text-xl:  1.7rem;  /* 최대 디스플레이 (카드 이모지·큰 가격) */
```

각 단계가 **어떤 역할을 담는지**를 주석으로 못박는다 — 스케일은 크기 목록이 아니라 역할의 그릇이다.

### 스냅 매핑 (현재 값 → 토큰; 이동 root 16px 기준)

| 토큰 | 값 | 흡수 (현재 font-size) | 최대 이동 |
|---|---|---|---|
| `--text-2xs` | 0.7rem | 0.68·0.72·0.74 | −0.6px (0.74) |
| `--text-xs` | 0.8rem | 0.75·0.78·0.79·0.8·0.82 | +0.8px (0.75) |
| `--text-sm` | 0.9rem | 0.85·0.88·0.9 | +0.8px (0.85, 4곳) |
| `--text-md` | 1rem | 0.95·0.98·1·1.05 | +0.8px (0.95, 4곳) / −0.8px (1.05) |
| `--text-lg` | 1.5rem | 1.5(h1) · **1.3(‹ › 넘김 셰브론 → 의도적 상향)** | **+3.2px (‹ › 셰브론, 의도)** |
| `--text-xl` | 1.7rem | 1.7 | 0 |

- **‹ › 넘김 셰브론(1.3→1.5) 한 곳만 +3.2px, 나머지 이동은 전부 ≤0.8px.** 시그니처(0.9·1.5·1.7)와 1.0은 무이동.
- 이동이 몰리는 곳: 0.85 클러스터(4곳 → 0.9)·0.95 클러스터(4곳 → 1.0) 각 +0.8px.
- **값이 같은 토큰에 모여도 의도된 것이다.** 예: 아이브로(0.9)·필터 라벨(0.85)·nodrop(0.88)이 `sm`에
  모이지만, 역할 구분은 **서체·색·굵기**(안 건드리는 축)가 진다 — 크기 티어를 공유하는 게 맞다.
- **표에 없는 값이 나오면** 가장 가까운 단계로 스냅하되 이동 >2px면 리터럴 유지 + 한 줄 주석.
  (현재 감사에선 ‹ › 셰브론 1.3을 제외한 모든 rem 값이 ≤0.8px로 표에 있고, 1.3은 위처럼 의도적 상향.)

## 적용 원칙

- `src/style.css`의 `font-size: <N>rem` 선언 전부를 `var(--text-*)`로 교체.
- ‹ › 넘김 셰브론(`.nav`) `1.3rem`은 `--text-lg`(1.5)로 — 의도적 상향(위 결정). `.empty/.loading`은
  0.95→`md`로 평범한 스냅이다.
- **`font-size: 9px`(`.spark .val, .spark .lab`)는 그대로 둔다.** SVG 좌표계 데이터 라벨이다.
- `font-size` 외의 어떤 속성도 이 토큰으로 바꾸지 않는다(간격은 이미 `--space-*`).

**leading(line-height) 축을 미뤄도 안전한 이유(확인됨):** 이 파일의 모든 `line-height`가 **단위 없는
값**(1.45·1.5·1.55·1.65·1.7 등)이라 폰트 크기에 비례해 함께 늘어난다 — 크기만 바꿔도 줄 간격이
자동으로 맞는다. 고정 `line-height: 1`인 세 곳(카드 이모지·큰 가격·‹ › 셰브론)은 각각 1.7·1.7·(1.3→1.5)
로 **단일 줄**이라 줄바꿈 이슈가 없다. 그래서 이번에 leading을 안 건드려도 세로 리듬이 깨지지 않는다.

## 검증 (완료 게이트)

- **게이트 = `npm test` 와 `npx tsc --noEmit` 둘 다.** (CSS라 로직 영향 없지만 규칙.)
- **브라우저 재실측 + 스크린샷 사인오프** — 글자 크기는 이동이 보일 수 있어 이 단계가 핵심.
  메인(`/`) 헤더·필터·카드 리스트·카드 펼침(노트·스파크·레시피 칩·메모 **‹ › 넘김 셰브론 1.3→1.5, 유일한 큰 이동**)·`/coming`·빈 화면.
  특히 0.85→0.9, 0.95→1.0 클러스터가 몰린 곳(필터 라벨·취소선 가격·원단위·계절 칩)을 본다.
- **카드 타이틀 위계 확인(중요):** `.card-title` 1.05→`md`(1.0)이면 **본문(명시 크기 없어 1.0)과 같은
  크기**가 된다 — 제목의 크기 우위가 사라지고 **굵기(600)만으로** 위계를 진다. 브라우저에서 카드
  타이틀이 여전히 제목처럼 읽히는지 반드시 본다(안 읽히면 `.card-title`을 예외로 둘지 재검토).
- **회귀 없음:** 토큰화 전/후 비교로 의도한 이동 외 변화(줄바꿈 달라짐·오버플로) 없는지.
- **완결성 grep:** `grep -nE "font-size:[^;]*[0-9]\.?[0-9]*rem" src/style.css` → 리터럴 rem이
  남지 않아야 한다(9px는 별개). `grep`으로 미정의 `--text-*` 참조도 없는지 본다.

## 범위 밖 (하지 않는다)

- **`line-height`·`letter-spacing`·`font-weight`.** 별개 축. 크기 스케일이 자리 잡은 뒤 필요하면 별도.
- **`.spark`의 `9px`.** SVG 데이터 라벨.
- **색·간격·폰트패밀리·라운드.** 이미 토큰화됨.
