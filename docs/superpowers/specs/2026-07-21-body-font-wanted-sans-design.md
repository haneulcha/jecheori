# 본문 폰트 통합 — Wanted Sans 콘텐츠 서브셋 설계

> 2026-07-21 · 앱 전반의 타이포를 **Wanted Sans 하나**로 통일한다. 마루부리(디스플레이
> 세리프)를 완전히 제거하고, 손글씨(받아쓰기)만 유일한 캐릭터 폰트로 남긴다.
> 브라우저 시안 3라운드(세리프 세 얼굴 → 세리프 폐기·고딕 전환 → 캐릭터 고딕 세 후보
> 앱 전반 적용) 끝에 **Wanted Sans** 확정. 결정 경로는 아래 "시안 반복 기록".

## 왜

지금 타이포는 사실상 세 갈래인데 정돈돼 있지 않다:

- **마루부리**(디스플레이 세리프)는 `h1`·절기 아이브로(`.week`) **헤더 두 곳에만** 산다.
  스크롤해 카드로 들어가면 사라진다 — 앱의 심장인 카드엔 한 글자도 없다.
- **받아쓰기**(손글씨)는 카드 `.why` 한 줄. 이건 적정.
- **시스템 산세리프**(`Apple SD Gothic Neo` 등)가 카드 품목명·소제목·본문·가격·영양·
  레시피 **전부**를 싣는다 — 눈에 가장 많이 닿는 표면 전체가 "디자인 안 된 기본값"이고,
  기기마다 얼굴이 달라(맥/윈/안드) **정돈되지 않은** 인상을 준다.

즉 문제는 "폰트가 세 개라서"가 아니라 **캐릭터 세리프가 헤더에 갇히고 카드는 시스템
기본값**이라 정체성이 화면에서 증발하는 것. 세리프를 카드 콘텐츠에 얹으면(소제목
"고르는 법" 등) 기능적 소제목에 옷이 과해 튄다 — DESIGN.md 대원칙 "과감함은 시그니처
(쪽빛 라인아트)에만"과도 충돌한다.

**해법**: 손글씨만 남기고 **전부 하나의 캐릭터 고딕으로**. 차분한 고딕 데이터 위에
연필로 적은 손글씨 한 줄이 유일한 "사람의 손"으로 떠오르는 그림 — 담백·"창이지 거울
아님" 톤에 세리프 산재보다 더 맞는다. 손글씨의 역할도 오히려 강해진다.

## 확정형

### 폰트 역할

| 역할 | 폰트 | 사용처 |
|---|---|---|
| 앱 전반 | **Wanted Sans** (self-host, 콘텐츠 서브셋, OFL) | 아이브로·h1·품목명·소제목·본문·가격 등 **거의 전부** |
| 손글씨 | **학교안심 받아쓰기 L** (self-host, 서브셋, OFL) | 카드 `.why` "제철이의 한마디" 한 줄 — 유일한 캐릭터 폰트로 승격 |
| 모노 | 시스템 모노 스택 (`SF Mono` …) | SeasonStrip 월 숫자·`.num` tabular — 변경 없음 |

**마루부리는 완전히 제거**한다(`--font-display` 역할 삭제, `src/fonts/MaruBuri-*.woff2`
2벌 삭제 ≈ 865KB). 세리프는 앱에서 사라진다.

### 웨이트 맵 (4벌 — 앱이 실제 쓰는 값 전수)

**원칙: 기존 웨이트를 그대로 보존한다(페이스만 교체).** CSS 전수 조사 결과 앱은
`400/600/700/800`을 쓴다 — 이 넷을 서브셋해 어떤 슬롯도 조용히 가벼워지지 않게 한다.

| 웨이트 | 슬롯 (현행 그대로) |
|---|---|
| `400` Regular | 본문·기준선·영양·레시피 기본 텍스트 · 절기 아이브로(`.week`) |
| `600` SemiBold | 카드 품목명(`.cardTitle`) · `.wonu`·스파크풋·힌트명·내비 현재 등 기존 600 |
| `700` Bold | `h1` · 등락 칩 · 스파크 now · 제철띠 현재월 등 기존 700 |
| `800` ExtraBold | 큰 가격(`.big`, `--text-xl`) — 이 하나를 위해 800을 뺄 수 없다 |

`400/600/700`만 서브셋하면 `.big`(800)이 700으로 매칭돼 **머리기사 가격이 미세하게
가벼워진다** → 800 포함 필수.

**소제목 라벨 역할**: "고르는 법/보관/쓰임" 라벨(`Note` `.lbl`)을 카드의 기존 라벨
(`recipeLabel`: 넓은 자간·소형·muted)과 **같은 결로 통일**한다 — `letter-spacing` 넓힘
(`0.08em`) + `600`. 세리프를 카드에 얹지 않고도 소제목이 "디자인된 마이크로 라벨"로
읽히게. 색(ink 유지 vs muted)은 브라우저 실측으로 확정.

### 서브셋 self-host (손글씨 방식의 확장)

앱의 사용자향 한글은 전부 빌드 소스에 바운드돼 있다 — `produce.json` +
`season.ts` + 컴포넌트 정적 문구. 실측 결과 **고유 문자 ~600자(한글 음절 525)** 뿐.
가격이 CI JSON에서 오듯, 이 폰트도 콘텐츠를 진실의 원천으로 삼아 서브셋한다.

**payload 실측 (weight당 woff2)**:

| 방식 | weight당 | 4벌 합계 |
|---|---|---|
| 통짜 full(전체 한글) | ~700KB | ~2,800KB |
| **콘텐츠 서브셋(채택)** | ~53KB | **~216KB** |

제거할 마루부리(865KB)보다 **648KB 가볍다** — 앱 전체 캐릭터 폰트를 얻으면서 payload는 준다.

**서브셋 소스**: `produce.json`(`name`·`whyNow`·`howToPick`·`howToStore`·`howToUse`) +
`season.ts`(절기명) + 컴포넌트·라우트의 정적 UI 문구 + 숫자·기호(`0-9 . , % ~ − · 원 kg g ml L ₩ ( ) …`).
스크립트 `scripts/subset-body-font.sh`(손글씨 `subset-handwriting-font.sh`의 형제)가
텍스트를 모아 `pyftsubset`으로 3벌을 `woff2`로 뽑는다.

**폴백 체인**: `'Wanted Sans', 'Apple SD Gothic Neo', 'Malgun Gothic', 'Noto Sans KR', sans-serif`.
`--font-hand` 폴백의 `'MaruBuri'`도 이 고딕 체인으로 교체.

### 빌드 가드 (딱 하나의 리스크 대응)

본문 폰트라 새 음절이 폴백되면 **문장 중간에 다른 고딕이 튀어** 눈에 잘 안 띈다
(손글씨 한 줄과 달리 표면이 넓다). 그래서 조용한 폴백 대신 **시끄러운 실패**로 바꾼다:

- **커버리지 검사**(테스트 또는 스크립트): 빌드 소스에서 서브셋 woff2에 **없는 글자**를
  발견하면 **실패**시킨다. 새 품목·copy 편집으로 새 음절이 들어오면 빌드가 깨져
  `scripts/subset-body-font.sh` 재생성을 강제한다.
- 완료 게이트(`npm test` + `npx tsc --noEmit`) 철학과 정합 — "빠진 게 부재로 조용히
  나타나 나중에 놀라움이 되는" 것을 막는다.
- **손글씨에도 같은 가드를 확장한다.** 원래는 "폴백이 한 줄 장식이라 허용"으로 뒀으나,
  실제로 수산물·축산물 확장 때 손글씨 서브셋 재생성이 누락돼 **121자·whyNow 72줄이
  시스템 폰트로 조용히 폴백**됐다(이 전환에서 발견). 폴백 허용 가정이 실재 회귀를 낳았으므로,
  whyNow 글자 ↔ 손글씨 서브셋도 커버리지 검사에 포함해 "빠지면 실패"로 바꾼다.

### 손글씨 서브셋 재생성 (드리프트 수정)

현 `HakgyoansimBadasseugi-L.woff2`는 커버 225자인데 필요 345자 — **121자 누락**
(고등어·갈치·조기·삼치 등 신규 whyNow). `scripts/subset-handwriting-font.sh`를 **방식 변경
없이 재실행**해 커버리지를 콘텐츠에 맞춘다(원본 TTF는 스크립트가 OFL로 내려받음). 이
전환의 마무리 작업에 포함한다.

## 변경 파일

- `src/fonts/` — Wanted Sans 3벌(`400/600/700`) woff2 추가, `MaruBuri-*.woff2` 2벌 삭제.
- `src/global.css` — `@font-face`(마루부리 제거 · Wanted 3벌 추가), `--font-body`를
  Wanted 체인으로, `--font-display` 토큰과 그 참조(`.week`·`h1`) 제거→해당 슬롯에
  웨이트 직접 지정, `--font-hand` 폴백 교체.
- 소제목을 렌더하는 컴포넌트 CSS — 라벨 역할(웨이트·자간·크기·색) 적용.
- `scripts/subset-body-font.sh` — 신규(서브셋 생성). 커버리지 가드(테스트/스크립트) —
  본문(Wanted)과 손글씨(whyNow) 둘 다 검사.
- `src/fonts/HakgyoansimBadasseugi-L.woff2` — 재생성(121자 드리프트 수정, 방식 불변).
- `DESIGN.md` — 타이포그래피 표 개정, 결정 기록에 이 전환 추가.
- `CLAUDE.md` — 명령어 목록에 서브셋 스크립트 한 줄(손글씨 옆).

## 검증 (완료 게이트)

- `npm test` + `npx tsc --noEmit` 둘 다 통과. 커버리지 가드 포함.
- **브라우저 실측** — `npm run dev`로 메인·`/coming`을 **모바일 28rem 폭**에서 열어
  아이브로·h1·품목명·소제목·본문·가격·손글씨가 의도대로 앉는지, 폴백 없이 Wanted로
  렌더되는지 확인. 사용자향 시각 변경이므로 스크린샷 사인오프.
- **손글씨 회귀 확인**: 고등어·갈치·조기·삼치 등 신규 whyNow 카드에서 손글씨가 폴백 없이
  받아쓰기체로 렌더되는지 확인(재생성 후 커버리지 검사 0 누락).
- 마루부리 참조·파일이 완전히 사라졌는지 grep 확인.

## 비목표 (YAGNI)

- 모노 스택 교체 안 함(SeasonStrip·tabular 숫자는 시스템 모노 유지).
- 손글씨 폰트·서브셋 **방식**은 변경 안 함(재생성만 — 드리프트 수정, 위 참조).
- 색·레이아웃·모션 변경 없음 — 타이포 얼굴/역할만.
- 통짜 full·가변폰트·unicode-range split 도입 안 함(콘텐츠 서브셋으로 충분·더 가벼움).

## 시안 반복 기록

1. **세리프 세 얼굴**(마루부리/고운바탕/송명)을 카드 제목·소제목까지 넓혀 비교 →
   세리프를 카드 콘텐츠에 얹으면 소제목이 튀고, 송명은 개성이 내용을 덮음. 세리프 폐기.
2. **손글씨만 빼고 고딕**으로 방향 전환. 시스템 스택 vs 캐릭터 고딕(IBM Plex) 비교 →
   시스템 스택은 기기 편차로 "정돈 안 됨". 캐릭터 고딕 하나를 앱 전반에 self-host로 결정.
3. **캐릭터 고딕 세 후보 앱 전반 적용**(Pretendard/IBM Plex Sans KR/Wanted Sans) →
   Pretendard는 완성도 높지만 "기본값 티", IBM Plex는 개성↔가독 균형, **Wanted Sans가
   Pretendard급 완성도에 온기를 더해 이 앱(계절·손글씨·라인아트)에 가장 맞음** → 확정.
